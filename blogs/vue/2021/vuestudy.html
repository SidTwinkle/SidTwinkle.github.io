    # Vue2生命周期

    • beforeCreate() : Vue实例创建之前， 此时，Vue中的变量,响应式，事件,注入还未开始，无法对Vue进行操作

    • created(): 能够访问变量， 可进行--- 初始化状态或挂载到当前实例的一些属性

    • 有没有 规定el 属性， 如果规定了，正常向下执行， 如果没有规定 ，则等待 new Vue().mount("") 后继续执行

    • 有没有 规定template 属性， 如果规定了，就会优先编译template模板 ， 如果没有规定 ，则 将el 对应的部分 的outerHTML（包含当前节点的所有标签）

    • beforeMount:实例挂载前， $ef-->模板解析之前最后一次修改 模板节点

    • mounted: 拿到真正的DOM节点

      ----> 当data 发生改变时会触发 update阶段

    • beforeUpdate: 数据发生改变前，访问dom， 访问的是 更新前的dom 节点， 可以记录更新以前的状态

      ---> 虚拟dom创建 ，diff算法对比， 打上补丁..

    • updated : 获取更新后的dom 节点, 进行某些插件的初始化工作， 如轮播插件. 数据更新完，可以用在dom中.

    • beforeDestroy:

    new Vue({
        el:"",

        },
        mounted () {

            // 依赖于dom创建之后， 才进行初始化工作的插件（如轮播插件）

            },2000)

        },
        updated () {
           // 数据更新完成

        }
    })
</script>

    # 反向代理&别名

    开发中常遇到跨域问题， 域名端口号都不同，被同源策略阻止

    jsonp 是利用script不受浏览器同源策略管辖的原理来进行跨域，不安全， 不能使用post请求

    在vue.config.js中配置反向代理， 配置在自己的服务器， 让它转发这个请求， 服务器与服务器之间没有跨域

    module.export = {
    // 配置反向代理

        proxy: {
            '/dong':{
                target:'https://www.meizu.com/'   // 以魅族官网未为例
                changeOrigin: true,
                pathRewrite: {     // 路径重写
                '/dong': ''
            }
            }
        }
    }
}

    # 路由引入

    SPA：

    • 单页面应用，由一个外壳和多个页面片段组成
    • 资源共用(css, js)：共用，只在外壳部分加载
    • 刷新方式：页面局部刷新或更改
    • url模式： a.com/#/pageone a.com/#/pagetwo
    • 用户体验: 片段切换快，用户体验好
    • 转场动画：容易实现
    • 数据传递：容易
    • 搜索引擎优化（SEO）：需要单独方案，实现较为困难，不利于（SEO）检索，可利用服务器端渲染（SSR）优化
    • 适用范围：高要求的体验度、追求界面流畅的应用
    • 开发成本：较高、常需借助专业的框架
    • 维护成本：相对容易

    MPA：

    • 多页面应用，由多个完整页面组成
    • 资源共用(css, js)：不共用，每个页面都需要加载
    • 刷新方式：整页刷新
    • url模式： a.com/pageone.html b.com/pagetwo.html
    • 用户体验: 页面加载缓慢，流畅度不够
    • 转场动画：无法实现
    • 数据传递：依赖url传参，或者cookie、localStorage等
    • 搜索引擎优化（SEO）：实现方法简易
    • 适用范围：追求高度支持搜索引擎的应用
    • 开发成本：较低、 但页面重复代码多
    • 维护成本：相对复杂

export default {
    setup() {
        const obj = reactive({
            list:[],
            text:''
        })
        
        const filterList = () => {
            return obj.list.filter(item => item.includes(obj.text))
        }

    }
}
